*for="expression"

Render an element for each iteration.

  • Expression can either be:
    • Any syntax supported inside for, for...in and for...of loops.
      • There is currently no distinction between let, const and var, but future versions may treat them differently so it is advised to follow best practices for declaring variables in JavaScript.
    • Any iterable object that implements Symbol.iterator.
      • Iterated key will be exposed as $key variable.
      • Iterated value will be exposed as $value variable.
    • A finite number.
      • Directive will be applied the specified number of times.
     string Evaluated value of the *id directive if present, or the identifier auto-generated by the directive. number Total number of iterations. number Current iteration index (0 indexed). number Current iteration index (1 indexed, same as $i + 1). number Whether this is the first iteration (same as $i === 0). number Whether this is the last iteration (same as $i === ($iterations - 1)).     *id="expression"

    Hint for *for directive to differentiate generated elements.

  • Must be defined on an element with a *for directive. Identifier must be unique within the loop, any duplicate is replaced by the last occurrence found.
