~test="expression"

Special directive for testing purposes.

Intended for testing purposes only. This is provided for developers that want to test their custom directives in isolation, without having to rely on other directives. string Any existing Phase name (e.g. ~test[testing], defaults to Phase.TESTING). Directive will be executed during the specified phase before any other directive of said phase, which can be used to simulate specific scenarios. Set element's textContent with expression result. Evaluate a JavaScript expression in the context of the element. Change the element to a Comment if expression is truthy (and revert it back otherwise).
